game_watcher.py
기존 game_monitor.py + live_data_watcher.py 파일 병합.(두 파일 다 원본은 존재)

 역할
실행 → 게임 감지 대기 → 게임 시작 감지 → 폴링 시작 → 게임 종료 감지 → 대기로 복귀
1. 게임 자동 감지
game_monitor.py의 run()을 내부 엔진으로 사용합니다. 1초마다 Riot Live Client API(https://127.0.0.1:2999)에 요청을 보내서 게임이 켜졌는지 확인하고, 켜지면 자동으로 수집을 시작합니다.
2. 게임 시작 시 (1회)
on_game_start 콜백이 실행되어 세션 상태를 초기화하고 콘솔에 소환사명, 내 팀, 아군/적군 챔피언, 게임 모드를 출력합니다.
3. 매 1초마다 폴링
on_poll 콜백이 실행되어 현재 게임 시간, 내 체력 비율, 적군 생사 상태를 수집합니다. 수집한 데이터는 session_log 리스트에 쌓이고 동시에 JSON 파일로 저장됩니다.
4. 게임 종료 시 (1회)
on_game_end 콜백이 실행되어 세션 전체를 JSON으로 저장하고 임시 파일을 정리한 뒤 대기 상태로 돌아갑니다.
5. 단축키 토글 (Ctrl+Shift+Z)
게임 풀스크린 중에도 수집을 중지/재시작할 수 있습니다. 재시작 시 build_callbacks()를 새로 호출해서 세션 상태가 깔끔하게 초기화됩니다.
6. 완전 종료 (Ctrl+C)

*블루 / 레드 기준으로 ORDER / CHAOS 구분 구현.
