# 새 노트북 이전 가이드

> **전제:** 새 노트북에 Miniconda, Claude (Cowork), League of Legends가 이미 설치되어 있어야 합니다.

---

## STEP 1 — 현재 노트북에서: 작업 내용 GitHub에 올리기

새 노트북으로 넘어가기 전에 **현재 변경사항을 전부 push** 해야 합니다.

```bash
# 프로젝트 폴더로 이동
cd C:\Users\USER\github\LoL_Realtime_Coach

# 변경된 파일 전체 스테이징
git add -A

# 커밋 (메시지는 원하는 대로)
git commit -m "chore: 노트북 이전 전 작업 내용 저장"

# 현재 브랜치를 push (현재 브랜치: feature/poc-riot-api)
git push origin feature/poc-riot-api
```

> **확인:** GitHub 저장소 페이지(https://github.com/kisoo98/LoL_Realtime_Coach)에서 커밋이 올라갔는지 확인하세요.

---

## STEP 2 — 새 노트북에서: 코드 받아오기

```bash
# 원하는 위치에 클론 (예: C:\Users\<이름>\github\)
git clone https://github.com/kisoo98/LoL_Realtime_Coach.git
cd LoL_Realtime_Coach

# 작업 브랜치로 전환
git checkout feature/poc-riot-api
```

---

## STEP 3 — 새 노트북에서: Conda 환경 만들기

```bash
# Python 3.10 환경 생성
conda create -n lol-coach python=3.10 -y

# 환경 활성화
conda activate lol-coach
```

---

## STEP 4 — 새 노트북에서: 패키지 설치

```bash
# 프로젝트 폴더 안에서
pip install -r requirements.txt
```

> **참고:** `dxcam`은 Windows DirectX 캡처 라이브러리로, Windows에서만 정상 설치됩니다. 설치 중 오류가 나면 건너뛰고 `mss`(2순위 폴백)로 동작합니다.

---

## STEP 5 — 새 노트북에서: API 키 설정

`.env.example`을 복사해서 `.env`를 만들고 API 키를 채워넣습니다.

```bash
copy .env.example .env
```

`.env` 파일을 열어서 아래 두 키를 입력하세요:

```env
# Grok (xAI) - https://console.x.ai/
XAI_API_KEY=실제_키_입력

# Gemini (Google AI Studio) - https://aistudio.google.com/app/apikey
GEMINI_API_KEY=실제_키_입력
```

> **현재 기본값:** `configs/config.yaml`의 `llm_provider`가 `"gemini"`로 설정되어 있으므로 **GEMINI_API_KEY가 필수**입니다.

---

## STEP 6 — Claude Cowork 연결 (선택)

Claude Cowork에서 이 폴더를 workspace로 연결하면 AI 어시스턴트와 함께 작업할 수 있습니다.

1. Claude 앱 실행 → Cowork 모드 진입
2. 폴더 선택 → `LoL_Realtime_Coach` 폴더 지정
3. 이전과 동일하게 대화 이어나가기

---

## STEP 7 — 실행 테스트

```bash
conda activate lol-coach
cd LoL_Realtime_Coach

# 메인 실행 (리그 오브 레전드 실행 중이어야 함)
python main.py

# 또는 Riot API PoC 테스트 (게임 실행 중이어야 함)
python live_data_watcher.py
```

> **단축키:** `F9` (기본값) 로 코칭 피드백 트리거

## 자주 발생하는 문제

**`dxcam` 설치 실패**
- Windows 전용 라이브러리입니다. 오류 메시지가 나와도 `mss`로 자동 폴백되니 무시해도 됩니다.

**`keyboard` 라이브러리 오류 (권한 문제)**
- Windows에서 `keyboard` 패키지는 관리자 권한이 필요할 수 있습니다.
- 터미널을 **관리자 권한으로 실행**하거나, `python main.py`를 관리자 권한으로 실행하세요.

**`GEMINI_API_KEY` 관련 RuntimeError**
- `.env` 파일에 `GEMINI_API_KEY`가 올바르게 입력되었는지 확인하세요.
- 무료 키는 https://aistudio.google.com/app/apikey 에서 발급받을 수 있습니다.

**YOLO 모델 파일 없음**
- `models/lol_minimap_yolo26n.pt` 파일이 `.gitignore`에 포함되어 있을 경우 GitHub에서 내려받지 못합니다.
- 현재 노트북의 `models/` 폴더를 USB 또는 클라우드로 직접 복사하세요.
